# 페이지 히트, 페이지 미스

## 페이지 히트
* 프로세스가 요청한 페이지가 이미 메모리에 적재되어 있는 상황
* 바로 접근 가능
## 페이지 미스
* 프로세스가 요청한 페이지가 메모리에 없는 상황
## 페이지 미스와 페이지 폴트
* 페이지 미스가 발생되면 페이지 폴트가 발생할 수 있음
* 그렇다고 무조건 페이지 폴트가 발생하는 건 아님
* 페이지 폴트가 발생했으면 무조건 페이지 미스가 발생

## 페이지 미스가 나지만 페이지 폴트로 이어지지 않는 상황
1. 접근 권한 오류
* 해당 페이지에 대해 접근 권한이 없거나 잘못된 접근이 시도되었을 경우
2. 성능 최적화를 위해 미리 로드할 때의 페이지 미스
* 성능 최적화를 위해 페이지를 실제로 사용하기 전에 **미리 요청**(**프리페칭**)하는 경우
* 실제 사용 전에 다른 데이터로 덮어쓰게 될 경우 페이지 미스는 발생하지만, 실제 사용자가 사용하지는 않아 페이지 폴트로는 발전하지 않음
